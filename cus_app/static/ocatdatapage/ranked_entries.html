How To Change The Same Parameter Values In Multiple ObsIDs: Ranked Entries for Constraints

Changing values in the ranked entries need careful consideration.

  • There are Constraint sections (Time Constraints, Roll Constraints, and ACIS Window Constraints) where ticking the box as "Yes" and then clicking "Update" will present new fields of "Ranked" entries.

  • If a new rank is added in the main ObsID, a new row of data is added to all secondary ObsIDs regardless of how many ranks they have.

    For example, consider the case that the main ObsID has one time constraint row, and a secondary obsid has three time constraint rows. If you add one new rank on the main ObsID, the the rank of the main one is changed from 1 to 2, and the rank of the secondary ObsID changes from 3 to 4, and the new data set will be in the rank 4 row.

  • If one of the rank parameter values in the main ObsID is modified:
    • if a secondary ObsID has the same values for all parameters in the same rank as the main ObsID, the value of the parameter will be updated.
    • if a secondary ObsID has a different value in the same rank, no change will be made.

    For example, if the main and a secondary ObsIDs have the ACIS window constraint values of:

      Ordr Chip Start Row Start Column Height Width Lowest Energy Energy Range Sample Rate
    main ObsId 1 I3 874 864 150 150 0.08 13 1
      2 I3 1 1 1024 1024 0.08 13 0
    Secondary ObsId 1 I3 874 864 150 150 0.08 13 1
      2 I3 1 1 1024 1024 0.08 13 0

    Then the change made in the main ObsID will also change the secondary ObsID. However, if they are different:

      Ordr Chip Start Row Start Column Height Width Lowest Energy Energy Range Sample Rate
    main ObsId 1 I3 874 864 150 150 0.08 13 1
      2 I3 1 1 1024 1024 0.08 13 0
    Secondary ObsId 1 I3 874 864 160 160 0.08 13 1
      2 I3 1 1 1024 1024 0.08 13 0

    Then the change made on the first rank of the main ObsID won't be made in the secondary ObsID. You can still make a change on the second rank in the main ObsID, which will be propagated to the secondary obisd as all parameter values are identical in both obsids.

  • If you remove one row from the ranked entries in the main ObsID, the same rank in a secondary ObsID will be removed only when all parameter values in the same rank are identical to those of the main ObsID. If they are not, no row from the secondary ObsID will be removed.
